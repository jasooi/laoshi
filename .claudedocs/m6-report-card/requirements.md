# Milestone 6: Report Card Dashboard -- Requirements Document

## Feature Overview

Milestone 6 replaces the placeholder Progress page (`/progress`) with a full **Report Card** dashboard. It delivers topline practice metrics, a daily sentences chart, AI-generated holistic teacher feedback, and a score breakdown with template-based descriptions.

**What already exists:**
- Frontend `Progress.tsx` with an empty placeholder ("Progress dashboard coming soon...").
- Backend `GET /api/progress/stats` returns home page stats (words_practiced_today, mastery_percentage, words_ready_for_review, total_words).
- `SessionWord` model stores averaged grammar_score, usage_score, naturalness_score per word per session.
- `SessionWordAttempt` model stores per-attempt scores with `created_ds` timestamp and `is_correct`.
- `UserSession` model stores session_start_ds, session_end_ds, summary_text.
- `UserProfile` model (1:1 with User) exists with encrypted API key columns.
- Three AI agents: orchestrator (Gemini), feedback (DeepSeek), summary (Gemini). BYOK support via `build_agents()`.
- mem0 stores cross-session user preferences and learning patterns.
- `SessionSummary.tsx` component shows end-of-session summary with "Back to Home" link.
- Sidebar links to `/progress` with "Progress" label and bar chart icon.

**What this milestone delivers:**
1. **Schema change**: New `report_card_feedback` Text column on `UserProfile`.
2. **Report Card AI agent**: New Gemini-based agent that generates holistic teacher feedback from mem0 memories + recent session summaries + rolling scores.
3. **Business logic service**: Topline metrics, daily chart data, rolling score averages, score descriptions, feedback generation.
4. **Two new API endpoints**: `GET /api/progress/report-card` (aggregated report card data) and `POST /api/progress/generate-feedback` (triggers AI feedback generation).
5. **Frontend rewrite**: Recharts stacked bar chart, topline metric cards, tabbed Teacher Feedback / Score Breakdown section, empty states.
6. **Integration**: Fire-and-forget feedback generation triggered from SessionSummary "Back to Home" click.

This milestone maps to **PRD User Story**: #16 (Detailed progress dashboard).

## User Stories

### US-01: View my Report Card
**As a** learner, **I want** to see a Report Card page with my practice statistics **so that** I understand my overall learning progress.

**Acceptance Criteria:**
- The page header shows a bar chart icon and "Report Card" title.
- Three topline metric cards display: Time Practiced (hours), Sessions Completed (count), Words Practiced (distinct count).
- Time Practiced sums session durations (session_end_ds - session_start_ds) across all completed sessions, capping each individual session at 2 hours.
- If total time is less than 1 hour, display in minutes (e.g. "45 min"). If 1 hour or more, display in hours (e.g. "1.5 hrs").
- Sessions Completed counts sessions with a non-null session_end_ds.
- Words Practiced counts distinct word_ids from completed SessionWords (status=1) across all sessions.
- If no sessions exist, all metrics show 0 or appropriate empty state.

### US-02: View daily practice chart
**As a** learner, **I want** to see a chart of my daily sentence attempts **so that** I can track my practice consistency and accuracy.

**Acceptance Criteria:**
- A stacked bar chart shows the last 7 days of practice data.
- Green bars represent correct sentences, red bars represent incorrect sentences.
- X-axis shows date labels in DD/MM format.
- Y-axis shows sentence count.
- Days with no practice show zero-height bars (always exactly 7 entries).
- Data comes from `SessionWordAttempt` grouped by date.
- If no data exists in the last 7 days, chart area shows "No practice data in the last 7 days".

### US-03: View teacher feedback
**As a** learner, **I want** to read holistic feedback from Laoshi about my overall learning **so that** I get personalised guidance beyond per-sentence evaluation.

**Acceptance Criteria:**
- The "Teacher Feedback" tab shows Laoshi's avatar (laoshi-logo.png) on the left and italicised feedback text on the right (no quotation marks).
- Feedback is generated by a dedicated Report Card AI agent using mem0 memories, recent session summaries, and rolling average scores.
- Feedback is 2-3 sentences in Laoshi's sassy-but-encouraging voice.
- A signoff line "-- Laoshi" is displayed at the bottom-right of the feedback area, with the seal image (`frontend/src/assets/seal.png`) displayed next to it as a stamp.
- Feedback is generated asynchronously when the user clicks "Back to Home" from the session summary (fire-and-forget).
- The generated feedback is stored in `UserProfile.report_card_feedback` (overwritten each time).
- If no feedback exists yet, the tab shows "Complete a session to get Laoshi's feedback!"

### US-04: View score breakdown
**As a** learner, **I want** to see my average scores for grammar, usage, and naturalness with explanations **so that** I know my specific strengths and weaknesses.

**Acceptance Criteria:**
- The "Score Breakdown" tab shows 3 score cards side by side: Grammar, Usage, Naturalness.
- Each card shows: a purple icon, a large score out of 10, the score name, an (i) info button, and a description.
- Icons: Grammar = document/notepad icon, Usage = speech bubble icon, Naturalness = smiley face icon.
- Scores are rolling averages from the user's last 5 completed sessions (falls back to all sessions if fewer than 5).
- Descriptions are template-based, mapped to score ranges (1-3, 4-5, 6-7, 8-9, 10) with specific text per score type.
- The (i) info button toggles a tooltip explaining what the score measures:
  - Grammar: word order, particles, verb aspect, measure words
  - Usage: correct meaning, context, collocations
  - Naturalness: native-like expression, idiomatic usage
- If no scores exist, tab shows "Complete a practice session to see your scores."

### US-05: Report Card agent respects BYOK
**As a** learner who uses custom API keys, **I want** the Report Card feedback generation to use my Gemini key **so that** feedback works with my own API quota.

**Acceptance Criteria:**
- `generate_report_card_feedback()` checks the user's `UserProfile` for a custom Gemini key.
- If a custom key exists, it builds a custom report card agent using that key.
- If no custom key exists (or decryption fails), the default agent is used.

## Functional Requirements

### Schema -- Backend

**FR-001**: `UserProfile` MUST have a new `report_card_feedback` column (Text, nullable). Default: null.

**FR-002**: An Alembic migration MUST add the column. No data migration needed (nullable column).

### Report Card AI Agent -- Backend

**FR-003**: A new `ReportCardContext` dataclass MUST exist in `ai_layer/context.py` with fields: `user_id` (int), `preferred_name` (str), `mem0_preferences` (str | None), `recent_summaries` (str | None), `avg_grammar` (float), `avg_usage` (float), `avg_naturalness` (float).

**FR-004**: A new `build_report_card_prompt()` function MUST exist in `chat_agents.py`. It MUST instruct the agent to write a 2-3 sentence feedback quip in Laoshi's voice (sassy, encouraging, specific). Inputs: mem0 memories, recent session summaries, rolling average scores. Output: JSON `{"feedback": string}`.

**FR-005**: A new `report_card_agent` MUST be defined using Gemini model (same as summary_agent). No tools or handoffs needed.

**FR-006**: A new `build_report_card_agent(gemini_api_key=None)` function MUST exist for BYOK support. If no custom key, return the default module-level agent. If custom key provided, build a new agent with custom Gemini client.

### Business Logic -- Backend

**FR-007**: `report_card_service.py` MUST contain `get_topline_metrics(user_id)` returning `{time_practiced_hours, sessions_completed, words_practiced}`.
- `time_practiced_hours`: SUM of (session_end_ds - session_start_ds) for completed sessions, with each individual session capped at 2 hours maximum, converted to hours, rounded to 1 decimal.
- The frontend MUST display the Time Practiced metric in minutes (e.g. "45 min") if the total is less than 1 hour, or in hours (e.g. "1.5 hrs") if 1 hour or more.
- `sessions_completed`: COUNT of UserSession where session_end_ds IS NOT NULL.
- `words_practiced`: COUNT DISTINCT word_id from SessionWord where status=1, joined to user's sessions.

**FR-008**: `get_daily_chart_data(user_id)` MUST return a list of `{date, correct, incorrect}` for the last 7 days.
- Query `SessionWordAttempt` grouped by DATE(created_ds), joined through SessionWord and UserSession to filter by user.
- SUM `is_correct=true` as correct, `is_correct=false` as incorrect.
- MUST fill missing days with `{date, correct: 0, incorrect: 0}` (always exactly 7 entries).
- Dates in ISO format (YYYY-MM-DD).

**FR-009**: `get_rolling_scores(user_id)` MUST return `{grammar, usage, naturalness}` as float averages.
- AVG scores from SessionWord where status=1, limited to the user's last 5 completed sessions (by session_end_ds DESC).
- If fewer than 5 completed sessions exist, use all completed sessions.
- Returns `{grammar: None, usage: None, naturalness: None}` if no completed SessionWords with scores exist.

**FR-010**: `get_score_description(score_type, score)` MUST return a descriptive string based on score range and type.
- Score ranges: 1-3 (needs work), 4-5 (developing), 6-7 (solid/good), 8-9 (strong), 10 (excellent).
- Each range has type-specific descriptions (grammar descriptions differ from usage descriptions).
- Returns None if score is None.

**FR-011**: `generate_report_card_feedback(user_id)` MUST:
- Fetch mem0 memories for the user.
- Fetch the last 3 session summaries (UserSession.summary_text, ordered by session_end_ds DESC).
- Fetch rolling scores via `get_rolling_scores()`.
- Build `ReportCardContext` and run the report card agent (with BYOK support).
- Parse the JSON response using the existing `_parse_json_from_string()` utility from `practice_runner.py` and store `feedback` in `UserProfile.report_card_feedback`.
- On agent failure, store a static fallback message.

### API Endpoints -- Backend

**FR-012**: `GET /api/progress/report-card` MUST be JWT-protected and return the following shape (all values are computed from the database and AI agent -- see FR-007 through FR-011 for calculation logic):
```json
{
  "topline": {"time_practiced_hours": "<float>", "sessions_completed": "<int>", "words_practiced": "<int>"},
  "chart_data": [{"date": "<YYYY-MM-DD>", "correct": "<int>", "incorrect": "<int>"}, "... (always 7 entries)"],
  "score_breakdown": {
    "grammar": {"score": "<float|null>", "description": "<string|null from get_score_description()>"},
    "usage": {"score": "<float|null>", "description": "<string|null from get_score_description()>"},
    "naturalness": {"score": "<float|null>", "description": "<string|null from get_score_description()>"}
  },
  "teacher_feedback": "<string|null from UserProfile.report_card_feedback>"
}
```

**FR-013**: If no data exists (new user), `GET /api/progress/report-card` MUST return:
```json
{
  "topline": {"time_practiced_hours": 0, "sessions_completed": 0, "words_practiced": 0},
  "chart_data": [{"date": "...", "correct": 0, "incorrect": 0}, ...],
  "score_breakdown": {
    "grammar": {"score": null, "description": null},
    "usage": {"score": null, "description": null},
    "naturalness": {"score": null, "description": null}
  },
  "teacher_feedback": null
}
```

**FR-014**: `POST /api/progress/generate-feedback` MUST be JWT-protected, trigger `generate_report_card_feedback(user_id)`, and return `{"feedback": "..."}` on success.

**FR-015**: Both endpoints MUST be registered in `app.py`.

### Frontend -- Types

**FR-016**: `types/api.ts` MUST include:
- `ReportCardTopline`: `{time_practiced_hours: number, sessions_completed: number, words_practiced: number}`
- `DailyChartData`: `{date: string, correct: number, incorrect: number}`
- `ScoreDetail`: `{score: number | null, description: string | null}`
- `ScoreBreakdown`: `{grammar: ScoreDetail, usage: ScoreDetail, naturalness: ScoreDetail}`
- `ReportCardData`: `{topline: ReportCardTopline, chart_data: DailyChartData[], score_breakdown: ScoreBreakdown, teacher_feedback: string | null}`

### Frontend -- API Helpers

**FR-017**: `progressApi` in `lib/api.ts` MUST have:
- `getReportCard()` -> `GET /api/progress/report-card`
- `generateFeedback()` -> `POST /api/progress/generate-feedback`

### Frontend -- Report Card Page

**FR-018**: `Progress.tsx` MUST be rewritten to show the Report Card layout:
- Header: bar chart icon + "Report Card" title.
- Topline metrics: 3 cards in a row (Time Practiced, Sessions Completed, Words Practiced).
- Daily chart: Recharts stacked BarChart with green (correct) and red (incorrect) bars.
- Tab section: "Teacher Feedback" and "Score Breakdown" tabs.

**FR-019**: The page MUST call `progressApi.getReportCard()` on mount and populate all sections.

**FR-020**: Teacher Feedback tab MUST show laoshi-logo.png avatar on the left and italicised feedback text on the right (no quotation marks). A "-- Laoshi" signoff MUST be displayed at the bottom-right of the feedback area, with the seal image (`frontend/src/assets/seal.png`) rendered next to it as a stamp. Both images are imported from `frontend/src/assets/`.

**FR-021**: Score Breakdown tab MUST show 3 score cards with:
- Purple icons: Grammar = document/notepad, Usage = speech bubble, Naturalness = smiley face.
- Large score display (X/10).
- Score name label.
- (i) info button that toggles a tooltip explaining the score dimension.
- Description text from the template.

**FR-022**: Empty states MUST be handled:
- No sessions: full-page empty state with icon.
- No chart data in last 7 days: "No practice data in the last 7 days".
- No feedback: "Complete a session to get Laoshi's feedback!"
- No scores: "Complete a practice session to see your scores."

### Frontend -- Integration

**FR-023**: `SessionSummary.tsx` "Back to Home" link MUST call `progressApi.generateFeedback().catch(() => {})` on click (fire-and-forget, silently ignore errors) before navigating.

**FR-024**: `Sidebar.tsx` MUST rename the "Progress" label to "Report Card". Keep same `/progress` route and bar chart icon.

## Non-Functional Requirements

**NFR-001**: Recharts MUST be installed as a frontend dependency (`npm install recharts`).

**NFR-002**: The Report Card page MUST follow the existing Tailwind CSS design system (purple color scheme, card-based layout, consistent spacing).

**NFR-003**: The `POST /api/progress/generate-feedback` endpoint blocks server-side until AI generation completes and returns the result. The frontend MUST call this endpoint fire-and-forget (not awaiting the response), so the user is NOT blocked waiting for AI generation.

**NFR-004**: The report card agent MUST handle AI provider failures gracefully -- store a static fallback message rather than crashing.

**NFR-005**: Rolling score calculations MUST handle edge cases: no sessions, sessions with no scored words, fewer than 5 sessions.

**NFR-006**: Daily chart MUST always return exactly 7 entries (today and 6 days prior), filling missing days with zeros.

## Out of Scope

The following items are explicitly NOT part of Milestone 6:

1. **Saved sentences** (PRD story #14): Moved to Milestone 7, Phase 3.
2. **Weekly/monthly chart toggle**: Only last 7 days. Additional time ranges can be added later.
3. **Score history trend line**: Only current rolling averages. Historical trends are a future enhancement.
4. **Export/share report card**: No PDF or social sharing.
5. **Custom chart date range picker**: Fixed 7-day window.
6. **Real-time chart updates**: Data refreshes on page load only.

## Decisions Log

**DL-001**: Rolling window of last 5 sessions for score aggregation. Reflects current ability rather than punishing early mistakes. Falls back to all sessions if fewer than 5.

**DL-002**: Distinct words only for Words Practiced metric. Shows vocabulary breadth, not raw attempt count, which better represents learning progress.

**DL-003**: Template-based score descriptions rather than AI-generated. Instant, zero cost, zero latency. AI-generated descriptions would add complexity and API costs for minimal benefit.

**DL-004**: Separate Report Card AI agent (not reusing summary agent). Summary agent produces per-session summaries. Report Card agent produces holistic cross-session feedback incorporating mem0 memories and multiple session summaries.

**DL-005**: Fire-and-forget feedback generation on "Back to Home" click. Avoids blocking the user after a practice session. Feedback is ready by the time they next visit the Report Card page.

**DL-006**: Single `report_card_feedback` column on UserProfile. Stores only the latest feedback. History is unnecessary -- the teacher assessment should always reflect current performance.

**DL-007**: Recharts for charting. Most popular React charting library, good TypeScript support, built-in stacked bar chart component. Alternatives considered: Chart.js (requires wrapper), D3 (overkill for one chart).

**DL-008**: Score Breakdown icons match the mockup: Grammar = document/notepad (written structure), Usage = speech bubble (communication/context), Naturalness = smiley face (native-like expression). All rendered in purple to match the design system.
